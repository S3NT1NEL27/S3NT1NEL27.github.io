Part 3: Row Level Security (RLS) - The New Part
This is what makes Supabase powerful and secure. It's PostgreSQL's security feature.

The Problem: If your API is public, anyone can query any data. User A could read User B's private wisdom requests.

The Solution: RLS Policies You write SQL rules that say "who can see/edit what."

Example Policy:

sql
-- Users can only read their own requests
CREATE POLICY "Users see own requests"
ON wisdom_requests
FOR SELECT
USING (auth.uid() = user_id);
This means: When someone queries wisdom_requests, PostgreSQL automatically filters to show only rows where user_id matches the logged-in user's ID.

Why this matters:

Security happens at the database level
You don't write filtering code in your app
Even if someone hacks your API key, they can't see others' data
